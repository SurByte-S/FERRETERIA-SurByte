## 1. Alcance

Esta Política de Privacidad describe el tratamiento de datos personales realizado mediante la plataforma FERRETERÍA / LLAVE MAESTRA, sistema web de gestión comercial para ferreterías.

La plataforma permite administrar usuarios internos, clientes, productos, presupuestos, ventas, caja, cuenta corriente, stock y movimientos comerciales.

## 2. Datos que puede tratar la plataforma

Según el uso del sistema, pueden tratarse los siguientes datos:

- datos de acceso de usuarios internos: identificador, correo electrónico y sesión autenticada;
- datos de membresía interna: ferretería asociada, rol, estado activo y nombre visible;
- datos de clientes: nombre, teléfono, correo electrónico, domicilio y notas;
- datos comerciales: presupuestos, ventas, productos adquiridos, importes, formas de pago y saldos pendientes;
- datos de cuenta corriente: débitos, pagos, ajustes, importes, notas y fechas;
- datos de caja: apertura, cierre, efectivo esperado, efectivo contado, diferencia y observaciones;
- datos de productos: SKU, código de barras, nombre, descripción, marca, categoría, unidad, precio, costo, stock e imagen;
- registros técnicos necesarios para operar el sistema.

Se recomienda no cargar datos sensibles ni información innecesaria para la gestión comercial.

## 3. Finalidad del tratamiento

Los datos se tratan para:

- permitir el acceso de usuarios autorizados;
- identificar la ferretería a la que pertenece cada usuario;
- gestionar clientes, presupuestos, ventas y comprobantes internos;
- controlar stock y movimientos de inventario;
- administrar caja diaria y formas de pago;
- registrar saldos de cuenta corriente;
- consultar historial comercial;
- brindar soporte técnico, mantenimiento y seguridad;
- cumplir obligaciones legales, comerciales o administrativas cuando corresponda.

## 4. Base normativa

El tratamiento deberá realizarse conforme la normativa argentina aplicable, en especial la Ley 25.326 de Protección de Datos Personales, el Código Civil y Comercial de la Nación y, cuando corresponda, la Ley 24.240 de Defensa del Consumidor.

La plataforma no garantiza por sí sola el cumplimiento legal integral. El titular del comercio debe implementar las medidas administrativas, contractuales y legales necesarias.

## 6. Acceso a los datos

Podrán acceder a los datos:

- dueños, administradores, encargados y vendedores autorizados;
- usuarios internos con permisos de consulta;
- personal técnico autorizado para tareas de soporte o mantenimiento;
- proveedores de infraestructura necesarios para operar la aplicación;
- autoridades competentes, cuando exista obligación legal.

Cada usuario deberá acceder únicamente a la información necesaria para cumplir su función.

## 7. Seguridad

La plataforma utiliza autenticación de usuarios, cookies de sesión, roles internos y reglas de acceso por ferretería. La base de datos prevé aislamiento por tenant y permisos por rol.

Sin perjuicio de ello, el responsable del comercio deberá:

- asignar usuarios individuales;
- proteger contraseñas;
- revocar accesos no vigentes;
- limitar permisos;
- resguardar claves técnicas;
- realizar backups;
- evitar compartir credenciales;
- revisar periódicamente la información cargada.

No existe un sistema absolutamente inmune a fallas, errores humanos o incidentes de seguridad.

El titular del comercio deberá definir políticas de conservación, eliminación, respaldo y archivo conforme sus necesidades y la normativa aplicable.
